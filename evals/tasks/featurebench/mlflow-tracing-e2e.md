# Eval: FeatureBench E2E — MLflow Tracing System (Code Output)

## Source

- **Benchmark**: [FeatureBench](https://huggingface.co/datasets/LiberCoders/FeatureBench) (ICLR 2026)
- **Instance**: `mlflow__mlflow.93dab383.test_trace.17fde8b0.lv1`
- **Repo**: mlflow/mlflow
- **Difficulty**: 15 files changed, 42k char problem statement, 52k char patch
- **SOTA resolve rate**: 11% (Claude Opus 4.5 via Claude Code)

## Difference from design-only eval

The design-only eval (`mlflow-tracing.md`) tests decomposition quality with `output_mode=design`.
This eval tests **actual code generation and test passing** with `output_mode=code`.

Key differences:
1. `output_mode=code` — SOLVE agents write real Python files in isolated worktrees
2. REUNIFY agents merge code and resolve import conflicts
3. Success is measured by pytest F2P + P2P tests, not design quality markers
4. Requires a testbed setup (clone mlflow, checkout base commit, install deps)

## Prerequisites

Run testbed setup first:
```bash
./evals/run.sh fb:mlflow-tracing --skill fractal --dry-run
```

## Prompt

The agent prompt is generated by `setup-testbed.sh` and includes the full 42k-char problem
statement with all Interface Descriptions. It is written to:
```
/tmp/featurebench-testbed/<instance-id>/.featurebench/agent_prompt.md
```

The core invocation is:
```
/fractal decompose "<full-problem-statement>"

Config: output_mode=code, max_depth=3, max_children=6, solve_mode=parallel
```

## Expected behavior

1. **DECOMPOSE** phase produces the same 6-child tree as the design eval
2. **SOLVE** phase writes actual Python code in isolated worktrees:
   - Each SOLVE agent gets `isolation: "worktree"` (per SKILL.md)
   - Code is written to the target files listed in the meta.json
   - Each agent commits with `fractal({slug}):` prefix
3. **REUNIFY** phase merges worktree branches:
   - Resolves import conflicts across the 15 files
   - Ensures consistent Span/Trace/TraceData interfaces
   - Runs tests after merge
4. **Scoring**: pytest on F2P + P2P tests

## Grading

FeatureBench binary scoring:
- **RESOLVED**: ALL F2P tests pass AND ALL P2P tests pass
- **FAILED**: Any test fails

Additionally, we track fractal-specific metrics:
- Decomposition tree shape (should match design eval)
- Number of interface conflicts detected and resolved
- Worktree merge success rate
- Time to completion
